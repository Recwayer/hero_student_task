# Описание проекта

Данный репозиторий содержит реализацию набора алгоритмов для симуляции боя в пошаговой игре **Heroes Battle**.  
Проект реализован на языке **Java** и предназначен для демонстрации работы алгоритмов, а также анализа их
алгоритмической сложности.

В проекте реализованы:

- генерация армии;
- симуляция боя между двумя армиями;
- поиск подходящих юнитов для атаки;
- поиск кратчайшего пути к цели на игровом поле.

Код структурирован в соответствии с принятыми стандартами, покрыт модульными тестами и собран в JAR-файл:  
**`build/libs/hero_student_task.jar`**

# Реализованные алгоритмы и их сложность

## 1. Генерация армии (`GeneratePresetImpl`)

**Описание алгоритма:**

Алгоритм формирует армию на основе:

- списка доступных типов юнитов;
- максимального лимита очков.

Ограничения:

- суммарная стоимость армии не превышает лимит;
- не более 11 юнитов одного типа;
- используются только переданные типы юнитов;
- каждому юниту назначаются корректные координаты;
- все юниты создаются живыми.

**Алгоритмическая сложность:** O(n log n + m)

---

## 2. Симуляция боя (`SimulateBattleImpl`)

**Описание алгоритма:**

Алгоритм выполняет пошаговую симуляцию боя между двумя армиями до тех пор, пока одна из сторон полностью не потеряет
всех живых юнитов.

Особенности:

- корректная работа с армиями разного размера;
- обработка одновременной гибели юнитов;
- защита от бесконечных циклов;
- корректная работа при отсутствии целей.

**Алгоритмическая сложность:** O(n^2 log n)

---

## 3. Поиск подходящих юнитов для атаки (`SuitableForAttackUnitsFinderImpl`)

**Описание алгоритма:**

Алгоритм определяет список юнитов, доступных для атаки, в зависимости от стороны (левая или правая армия).

Учитывается:

- порядок рядов;
- направление атаки;
- наличие `null`-элементов;
- состояние юнита (жив / мёртв).

**Алгоритмическая сложность:** O(n)

---

## 4. Поиск пути к цели (`UnitTargetPathFinderImpl`)

**Описание алгоритма:**

Алгоритм ищет кратчайший путь от атакующего юнита к целевому юниту на двумерном игровом поле.

Особенности:

- движение по вертикали, горизонтали и диагонали;
- живые юниты считаются препятствиями;
- мёртвые юниты игнорируются;
- возврат пустого пути при невозможности достижения цели;
- проверка выхода за границы поля.

**Алгоритмическая сложность:** O((W * H) * log(W * H))

---

# Тестирование

Проект покрыт модульными тестами с использованием **JUnit**.

Тесты проверяют:

- корректные сценарии работы алгоритмов;
- граничные случаи;
- обработку `null` и пустых коллекций;
- устойчивость к ошибочным входным данным;
- отсутствие бесконечных циклов.

Запуск тестов:

```bash
./gradlew test
```

# Сборка проекта

Сборка JAR-файла выполняется командой:

```bash
./gradlew build
```

После сборки JAR-файл доступен по пути:
**`build/libs/hero_student_task.jar`**
